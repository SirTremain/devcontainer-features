# Dev Container Features

This repository publishes a dev container Feature for installing the OpenAI Codex CLI.

## `codex`

The `codex` Feature installs the Codex CLI in the image and configures runtime Codex state to use the developer's host `~/.codex` directory.

```jsonc
{
    "image": "mcr.microsoft.com/devcontainers/base:ubuntu",
    "features": {
        "ghcr.io/<owner>/devcontainer-features/codex:1": {}
    }
}
```

The Feature sets `CODEX_HOME` to `/codex-home` for container processes, wraps the installed `codex` command so callers without `CODEX_HOME` also default to `/codex-home`, and bind mounts `${localEnv:HOME}/.codex` to that path. Create the host directory before rebuilding the container if it does not already exist:

```bash
mkdir -p ~/.codex
```

Codex file-based auth, user config, logs, sessions, skills, and related state live under `CODEX_HOME`. The CLI binary and standalone package are installed outside `CODEX_HOME`, so the runtime mount does not hide the installed executable.

## VS Code Default Feature

To install this Feature automatically in dev containers opened with the VS Code Dev Containers extension, add it to your User `settings.json` as a default Feature:

```jsonc
{
    "dev.containers.defaultFeatures": {
        "ghcr.io/<owner>/devcontainer-features/codex:1": {}
    }
}
```

Replace `<owner>` with the GitHub owner that publishes this repository's GHCR packages. The same setting can include options if you want to pin the Codex CLI release installed by the Feature:

```jsonc
{
    "dev.containers.defaultFeatures": {
        "ghcr.io/<owner>/devcontainer-features/codex:1": {
            "version": "latest"
        }
    }
}
```

## Options

| Option | Description | Type | Default |
| --- | --- | --- | --- |
| `version` | Codex CLI release to install. Use `latest` or a release version such as `x.y.z`. | string | `latest` |

## Development

Each Feature lives under `src/<feature>` with a `devcontainer-feature.json` manifest and an `install.sh` entrypoint. Tests live under `test/<feature>`.

Run the Codex Feature tests with:

```bash
mkdir -p ~/.codex
devcontainer features test --skip-scenarios -f codex -i mcr.microsoft.com/devcontainers/base:ubuntu .
devcontainer features test -f codex --skip-autogenerated --skip-duplicated .
```

Package the local feature collection as a manifest validation check with:

```bash
devcontainer features package ./src --output-folder /tmp/devcontainer-features-output --force-clean-output-folder
```

## Publishing

The release workflow publishes Features from `./src` to GitHub Container Registry using `devcontainers/action`.
